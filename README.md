# SQLi-Feature-extraction

SQL injection detection codebase with a feature-based XGBoost method, multiple text-feature baselines, ablation experiments, and cross-dataset evaluation.

面向 SQL 注入检测的实验仓库，包含：
- `ours`：12 维手工特征 + XGBoost 主方法
- `fe/*`：BoW、TF-IDF、Word2Vec、FastText、BERT 基线
- `experiments/ablation`：手工特征消融实验
- `cross_dataset`：跨数据集训练与预测流程

All commands should be run from the repository root.

## Overview | 项目简介

This repository studies SQL injection detection from two angles:
- a compact handcrafted feature pipeline (`ours`)
- several feature-extraction baselines (`fe`)

The main method uses 12 structural, lexical, and semantic features extracted from SQL/query text, then trains an XGBoost classifier. The baseline folder provides alternative representations for comparison, including bag-of-words, TF-IDF, Word2Vec, FastText, and BERT.

本项目重点不是做一个通用 Web 防护系统，而是围绕 SQLi 检测任务复现实验、做方法对比，并分析手工特征是否有效、哪些特征更关键。

## Repository Layout | 目录结构

- `ours/`: main handcrafted-feature pipeline and trained artifacts
- `fe/`: baseline feature extractors and their train/predict scripts
- `experiments/ablation/`: ablation study for the 12 handcrafted features
- `cross_dataset/`: evaluation on external datasets with isolated outputs
- `results/`: aggregated CSV results and summary plots
- `Data/`: dataset inputs and train/test splits

## Data Format | 数据格式

Input files are expected to be CSV or JSON with:
- `Query`
- `Label`

`Label` can be either:
- `0` / `1`
- `normal` / `attack` (case-insensitive)

## Quick Start | 快速开始

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Split the default dataset

```bash
python ours/split_data.py
```

This generates:
- `Data/train_set.csv`
- `Data/test_set.csv`

### 3. Train the main model

```bash
python ours/train.py
```

Main artifacts:
- `ours/model/1/scaler_for_numeric.pkl`
- `ours/model/1/numeric_features/model_XGBoost.pkl`
- `ours/model/1/numeric_features/best_threshold.pkl`

### 4. Predict with the main model

```bash
python ours/predict.py
# or
python ours/predict.py Data/test_set.csv
```

Main outputs:
- `ours/model/1/numeric_features/predictions.csv`
- confusion matrices and ROC plots under `ours/model/1/numeric_features/`
- appended metrics in `results/csv/results_raw.csv`

### 5. Run baselines

Each baseline has the same pattern:

```bash
python fe/<method>/train.py
python fe/<method>/predict.py Data/test_set.csv
```

Supported methods:
- `bow`
- `tfidf`
- `w2v`
- `fasttext`
- `bert`

Example:

```bash
python fe/bow/train.py
python fe/bow/predict.py Data/test_set.csv
```

### 6. Post-process and plot results

```bash
python results/cal.py
python results/plot.py
```

### 7. Optional experiments

Run the handcrafted-feature ablation study:

```bash
python experiments/ablation/run_ablation.py
```

Run cross-dataset training and prediction for the currently supported external datasets:

```bash
python cross_dataset/train_external.py --dataset all
python cross_dataset/predict_external.py --dataset all
```

## FastText Artifact Note | FastText 文件说明

The file below is intentionally not included in this repository:

`fe/fasttext/model/1/fasttext.model.wv.vectors_ngrams.npy`

Reason:
- it is a very large sidecar artifact generated by `gensim` FastText
- the file size is too large to upload to GitHub comfortably in this repository

Impact:
- `fe/fasttext/predict.py` cannot directly run from the current repo state unless this file is present
- loading `fe/fasttext/model/1/fasttext.model` also depends on this sidecar file
- this means the FastText baseline is not fully reproducible from the checked-in artifacts alone

How to regenerate it locally:

```bash
python fe/fasttext/train.py
```

This will regenerate the FastText artifacts under `fe/fasttext/model/1/`, including:
- `fasttext.model`
- `fasttext.model.wv.vectors_ngrams.npy`
- `model_XGB.pkl`
- `train_metrics.pkl`

Important:
- `fasttext.model.wv.vectors_ngrams.npy` is not an optional attachment
- if it is missing, the saved FastText model cannot be loaded normally

简而言之：这个 `.npy` 文件不是“可删的附属文件”，而是当前 FastText 模型加载所需的配套产物；只是因为体积过大，没有一并上传到 GitHub。

## Notes | 补充说明

- `cross_dataset` currently supports the external datasets exposed by the checked-in scripts, and keeps their outputs separate from the main `Data/` and `ours/model/` artifacts.
- `experiments/ablation` is designed to answer whether the 12 handcrafted features are useful as a whole, which feature groups matter most, and what happens when a group or a single feature is removed.
- Some historical result files and plots are already checked in, so the repository can be inspected before full retraining.
